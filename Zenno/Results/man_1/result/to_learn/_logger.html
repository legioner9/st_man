    • ОБЗОР</p> logger [-is] [-f файл] [-p приоритет] [-t tag] [message ...]

     

    ОПИСАНИЕ

    Logger предоставляет командный интерфейс для модуля системного журнала syslog(3) .

    Опции:

    -i


          Заносить в журнал вместе с каждой строкой id протоколируемого процесса.

    -s


          Помещать message не только в журнал, но и на страндартное устройство ошибок.

    -f файл

    Вести протоколирование из данного файла.

    -p приоритет

    Ввести сообщение со заданным приоритетом. Приоритет может быть задан численно или как пара ``facility.level''. Например, ``-p local3.info'' пишет в журнал сообщение как info уровень в local3 facility. По умолчанию установлено ``user.notice.''

    -t тэг

    Помечает кажду строку в журнале заданным тэгом.

    сообщение

    Записывает сообщение в журнал; если оно не задано и не задан флаг -f, то протоколирование ведется из стандартного ввода.

    Утилита logger возвращает 0 при успешной работе и >0 при возникновении ошибок.

    ПРИМЕРЫ

    logger System rebooted

    logger -p local0.notice -t HOSTIDM -f /dev/idmc

     

    СМ. ТАКЖЕ

    syslog(3) , syslogd(8)

    СТАНДАРТЫ

    Ожидается, что команда logger будет совместимой с IEEE Std1003.2 (``POSIX'').
